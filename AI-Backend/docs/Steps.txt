========================================
 Ki-trav-chat — Development Server Setup
========================================

1. Navigate to the project directory
   cd /Users/ishaankurmi/Desktop/Uni/Sem_7/Capstone/Ki-trav-chat

2. Activate the virtual environment
   source .venv/bin/activate

3. Ensure .env is configured correctly
   - GOOGLE_APPLICATION_CREDENTIALS must point to the correct serviceAccountKey.json path:
     GOOGLE_APPLICATION_CREDENTIALS=/Users/ishaankurmi/Desktop/Uni/Sem_7/Capstone/Ki-trav-chat/serviceAccountKey.json
   - OPENAI_API_KEY, AMADEUS_API_KEY, AMADEUS_API_SECRET, and FIREBASE_WEB_API_KEY must be set

4. Start the Flask server
   python app.py
   → Runs on http://localhost:5001

5. Start ngrok with permanent static domain
   ngrok http --url=unnoticed-demeanor-ambiguity.ngrok-free.dev 5001
   → Permanent public URL: https://unnoticed-demeanor-ambiguity.ngrok-free.dev
   → Use https://unnoticed-demeanor-ambiguity.ngrok-free.dev/chat as the API URL in FlutterFlow

6. (Optional) Expose server to FlutterFlow via localtunnel (URL changes every restart)
   npx localtunnel --port 5001
   → Outputs a public URL like: https://xxxx-xxxx-xxxx.loca.lt
   → Use https://<your-tunnel-url>.loca.lt/chat as the API URL in FlutterFlow

7. (If using localtunnel) Add bypass header in FlutterFlow
   - In your API call headers, add:
     Bypass-Tunnel-Reminder: true
   - Without this header, requests will return 511

========================================
 Notes
========================================
- The ngrok static domain never changes — same URL every time you run step 5
- The localtunnel URL changes every time you restart it
- The Flask server runs in debug mode by default; do not use in production
- To kill existing server instances on port 5001:
  lsof -ti :5001 | xargs kill -9
